## OpenClaw runtime upgrade content

The production add-on now downloads official upstream `openclaw/openclaw`
release `2026.5.18` instead of using the old vendored runtime.

The first production CI attempt failed because browser extension dependencies
used `workspace:*` and `npm --prefix ... install` could not resolve them. The fix
was to use `pnpm` for browser extension production dependency installation.

## HAOS store issue

HAOS initially did not detect the update because Supervisor internet was false.
After network health was repaired and `ha store reload` was run, HAOS detected
`version_latest: 2026.05.20.1`, then later `2026.05.20.2`.
